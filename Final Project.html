1 Data Preprocessing¶

Before we load the data, we should preprocess the dataset, because the size of dataset is not big enough for training the model with best performance.

To prevent the original dataset being modified, we need to create an image_pre folder and a mask_pre folder to save the preprocessing data.

We flip the images and rotate the images within 15 degrees, because we noticed that all the MR images are upwards, only with different degrees, therefore, we just flip the images and rotate them with small degrees.

def pre_processing(path):
    """
    The function to preprocessing the dataset
    :param path: The root path of the training set.
    :return: True if successive
    """
    
    img_files = glob(os.path.join(path, 'image', '*.png'))  # search every paths of all images in the image folder
    mask_files = []  # mask_files is saving the paths of all masks
    
    # Save every masks path for corresponding images.
    for img_path in img_files:
        basename = os.path.basename(img_path)
        mask_files.append(os.path.join(path, 'mask', basename[:-4] + '_mask.png'))
        
    # Empty the image_pre and mask_pre folders.
    empty_folder(os.path.join(path, 'image_pre'))
    empty_folder(os.path.join(path, 'mask_pre'))
    
    # Instantiate the horizontal flip, because p=1, every image would be flipped horizontally.
    transformflp = transforms.RandomHorizontalFlip(p=1) 
    
    # For each image and its corresponding mask, we flip horizontally and rotate them within 15 degrees.
    for index in range(len(img_files)):
        
        # Open the image and mask.
        img = Image.open(img_files[index])
        mask = Image.open(mask_files[index])
        
        # Flip the image and mask.
        img_flp = transformflp(img)
        mask_flp = transformflp(mask)

        basename = os.path.basename(img_files[index])  # get the filename in order to save the files.
        
        # Rotate the image and mask with different degrees, and save them.
        for i in range(3):
            if i == 1:
                 continue
            img.rotate(15 - i * 15).save(os.path.join(path, 'image_pre', basename[:-4] + '_rotate_' + str(i) + '.png'))
            img_flp.rotate(15 - i * 15).save(
                os.path.join(path, 'image_pre', basename[:-4] + '_flip_rotate_' + str(i) + '.png'))
            mask.rotate(15 - i * 15).save(
                os.path.join(path, 'mask_pre', basename[:-4] + '_rotate_' + str(i) + '_mask.png'))
            mask_flp.rotate(15 - i * 15).save(
                os.path.join(path, 'mask_pre', basename[:-4] + '_flip_rotate_' + str(i) + '_mask.png'))

        # Save the original image and mask to the preprocessing folders. 
        img.save(os.path.join(path, 'image_pre', basename[:-4] + '.png'))
        img_flp.save(os.path.join(path, 'image_pre', basename[:-4] + '_flip.png'))
        
        # Save the flipped image and mask.
        mask.save(os.path.join(path, 'mask_pre', basename[:-4] + '_mask.png'))
        mask_flp.save(os.path.join(path, 'mask_pre', basename[:-4] + '_flip_mask.png'))
        
        # Equalize the histogram to make the comtrast more clear.
        equalizehist(path)
    return True

def equalizehist(path):
    """
    The function to equalize the histogram of the image.
    :param path: The folder where the images are.
    :return: True if successive
    """
    
    img_path_list = glob.glob(os.path.join(path, 'mask_pre','*.png'))
    for img_path in img_path_list:
        img = cv2.imread(img_files[index], cv2.IMREAD_UNCHANGED)
        img = cv2.equalizeHist(img)
        cv2.imwrite(img_path, img)
    
    return True

def empty_folder(path):
    """
    The function to empty the folder.
    :param path: The folder which you want to empty.
    :return: True if successive
    """
    
    # Save the paths of all images in this folder. 
    files = glob(os.path.join(path, '*.png'))
    for i in files:
        os.remove(i)
    
    return True

class TrainDataset(data.Dataset):
    def __init__(self, root='', train = True):
        super(TrainDataset, self).__init__()
        if train == True:
            img_folder = 'image_pre'
            mask_folder = 'mask_pre'
        else:
            img_folder = 'image'
            mask_folder = 'mask'
        self.img_files = glob(os.path.join(root, img_folder, '*.png'))
        self.mask_files = []
        for img_path in self.img_files:
            basename = os.path.basename(img_path)
          
            self.mask_files.append(os.path.join(root, mask_folder, basename[:-4] + '_mask.png'))

    def __getitem__(self, index):
        img_path = self.img_files[index]
        mask_path = self.mask_files[index]
        data = cv2.imread(img_path, cv2.IMREAD_UNCHANGED)
        data = np.array([data, data, data])
        label = cv2.imread(mask_path, cv2.IMREAD_UNCHANGED)
        return torch.from_numpy(data).float(), torch.from_numpy(label).float()

    def __len__(self):
        return len(self.img_files)

# Pretrain module
def pre_train(model, pretrain=False):
    """
    The function to use the pretrain weights.
    :param model: model to be added weights
    :param pretrain: True when pretrain applys
    :return: model added the pretrain weights
    """
    
    if pretrain:
        # We downloaded the resnet50 from the official website.
        # "deeplabv3_resnet50_coco": "https://download.pytorch.org/models/deeplabv3_resnet50_coco-cd0a2569.pth"
        weights_dict = torch.load("./deeplabv3_resnet50_coco.pth", map_location='cpu')

        # As we said before, the number of classes in this pretrain model is 21.
        # We need to delete the weights about the classifier to ensure the shape of the weights is same.
        if num_classes != 21:
            for k in list(weights_dict.keys()):
                if "classifier.4" in k:
                    del weights_dict[k]

        # Load the pretrain model.
        model.load_state_dict(weights_dict, strict=False)
    return model

2.3 Define a Loss Function¶

Here we used the Dice+CE loss function. Its dice function use Laplace smoothing, which is adding one on both numerator and denominator sides. It can reduce the probability of overfiting, because it can make the coefficient get greater and make the loss be smaller, therefore the convergence will be happened sooner.

In [40]:
import torch.nn as nn
import torch.nn.functional as F
class DiceCELoss(nn.Module):
    def __init__(self, weight=None, size_average=True):
        super(DiceCELoss, self).__init__()

    def forward(self, inputs, targets, smooth=1):
        """
        The forward function of Dice+CE loss.
        :param inputs: The mask which is produced by images.
        :param targets: The mask which is given.
        :param smooth: The smooth variable to prevent the denominator be zero 
                       and to reduce the probability of overfitting
        :return: the sum of main loss and aux loss.
        """
        
        losses = {}
        targets_view = targets.view(-1)  # expand the target to 1-dimension
        
        # for main classifier and aux classifier separately
        for name, x in inputs.items():

            # expland the input image to 1-dimension
            x_view = x.argmax(1)
            x_view = x_view.view(-1)
    
            dice_loss = 0.0
        
            # for each classes
            for i in range(4):
                tmp_1 = (x_view == i).float()  # find all pixals in input image which is belong to current class 
                tmp_2 = (targets_view == i).float()  # find all pixals in output image which is belong to current class 
                intersection = (tmp_1 * tmp_2).sum()  #  the intersection of tmp_1 and tmp_2
                dice_loss += 1 - (2. * intersection + smooth) / (tmp_1.sum() + tmp_2.sum()+ smooth)  # calculate the dice loss with smooth for this class
            dice_loss_avg = dice_loss / 4.  # calculate the average dice loss
            CE = F.cross_entropy(x, targets.long())  # calculate the cross-entropy
            Dice_CE = CE + dice_loss_avg  # sum of cross entropy and dice loss

            losses[name] = Dice_CE  # save to the current classifier
        
        # if there is only main classifier
        if len(losses) == 1:
            return losses['out']
        
        return losses['out'] + 0.5 * losses['aux']

def create_optimizer(model,init_lr, aux=True):
    """
    The function to create optimizer.
    :param model: model to be added weights
    :param init_lr: initial learning rate
    :param aux: True if using aux optimizer
    :return: optimizer
    """

    # Set optimizer parameters.
    params_to_optimize = [
        {"params": [p for p in model.backbone.parameters() if p.requires_grad]},
        {"params": [p for p in model.classifier.parameters() if p.requires_grad]}
    ]
    if aux:  # if use aux optimizer
        params = [p for p in model.aux_classifier.parameters() if p.requires_grad]
        params_to_optimize.append({"params": params, "lr": init_lr*10})

    # Use AdamW as our optimizer.
    optimizer = torch.optim.AdamW(
        params_to_optimize,
        lr=init_lr, weight_decay=1e-4
    )
    return optimizer

Use the current model to predict the masks of validatiation dataset and then calculate their overall dice and dices for separate classes.

In [45]:
def categorical_dice(mask1, mask2, label_class=1):
    """
    Dice score of a specified class between two volumes of label masks.
    (classes are encoded but by label class number not one-hot )
    Note: stacks of 2D slices are considered volumes.

    Args:
        mask1: N label masks, numpy array shaped (H, W, N)
        mask2: N label masks, numpy array shaped (H, W, N)
        label_class: the class over which to calculate dice scores

    Returns:
        volume_dice
    """
    
    mask1_pos = (mask1 == label_class).float()
    mask2_pos = (mask2 == label_class).float()
    dice = 2 * (mask1_pos * mask2_pos).sum() / (mask1_pos.sum() + mask2_pos.sum())
    return dice

def evaluate(model, data_loader, device, num_classes):
    """
    Predict the masks of validation datasets by using current model and calculate the overall dice and dices for separate classes.
    :param model: current model and weights
    :param data_loader: the data_loader should contain validation data
    :param device: device to run current programme
    :param num_classes: number of classes needed to be segmented 
    :return: overall dice and dices for each class
    """

    dice_avg = 0.
    dice_class = [0., 0., 0., 0.]

    model.eval()  # model evaluate mode, will not change the weight of the model

    # Stop gradient calculatation to accelerate the calculation and save video memory.
    with torch.no_grad():

        # for the images and corresponding masks of each batch in the validation dataloader
        for image, target in data_loader:
            image, target = image.to(device), target.to(device)  # move the work to the current device

            # output the prediction mask for current image
            output = model(image)
            output = output['out']

            output = output.argmax(1).squeeze(0)  # regularation to the same format with the original mask
            dice = 0.

            # for each of classes
            for i in range(num_classes):
                dice_class[i] += categorical_dice(output, target, i) # calculate the dice for each class

                # if the current class is not background, add its dice to the variable which will to be used to calculate the overall dice later.
                if i != 0:
                    dice += categorical_dice(output, target, i)
          
            dice /= 3.  # calculate the overall dice
            # print(dice)
            dice_avg += dice

    cnt = len(data_loader)
    dice_avg /= cnt  # get the overall dice
    for i in range(num_classes):
        dice_class[i] = dice_class[i].item() / cnt  # get the dice for each class
    return dice_avg, dice_class

from torch.utils.data import DataLoader
import datetime
def training(optimizer, lr_scheduler, epoch_num, batch_size, if_show=True):
    """
    Training the model with the training set.
    :param optimizer: the optimizer used
    :param lr_scheduler: the learning scheduler used
    :param epoch_num: maximum number of epoches
    :param batch_size: number of images in one batch
    :param if_show: show the print text or not, default: True
    :return: text file name
    """

    # Create a txt file to record the data.
    txt_name = "results{}.txt".format(datetime.datetime.now().strftime("%Y%m%d-%H%M%S"))  
    f = open(txt_name, 'w+')  # create the file


    # Set some variables for loading datas.
    num_workers = 0  # number of threads, set to 0 because it may crush when using cuda on Windows 

    # Load the training set.
    train_data_path = './data/train'
    train_set = TrainDataset(train_data_path, train= True)
    train_data_loader = DataLoader(dataset=train_set, num_workers=num_workers, batch_size=batch_size, shuffle=True)

    # Load the validation set.
    val_data_path = './data/val'
    val_set = TrainDataset(val_data_path, train= False)
    val_data_loader = DataLoader(dataset=val_set, num_workers=num_workers, batch_size=1, shuffle=True)

    
    # Start the training module.
    for epoch in range(epoch_num):
        
        loss_avg = 0.  # the variable to save the loss
        
        model.train()  # model to training mode
        
        if if_show:
            print('Epoch[{}] start:'.format(epoch))
        
        # for the images and masks of each batch in the training data loader
        for image, target in train_data_loader:
            image, target = image.to(device), target.to(device)
            
            # calculate the loss
            output = model(image)
            loss = loss_func(output, target)

            optimizer.zero_grad()  # Initial the gradient of optimizer to be zero.
            loss.backward()  # backward propagation 
            optimizer.step()  # update the weights of optimizer
            lr_scheduler.step()  # update the learning rate scheduler
            lr = optimizer.param_groups[0]["lr"]  # update the learning rate
            loss_avg += loss.item()  # add to the loss

        loss_avg /= 1. * len(train_data_loader.dataset) / batch_size  # calculate the average loss for each image
        
        # Calculate the validation overall dice and dice for each class.
        val_dice, val_dice_class = evaluate(model, val_data_loader, device, num_classes)
        
        print_txt = 'epoch[{}/{}], train loss={}, val dice={}, val dice class={}, lr={}'.format(epoch, epoch_num-1, loss_avg, val_dice,
                                                                                      val_dice_class, lr)
        if if_show:
            print(print_txt)
        f.write(print_txt + '\n')  # record the info into the text file
        
        # Save the model
        save_file = {  
          "model": model.state_dict(),
          "epoch": epoch,
        }
#         torch_path = r'./models/model_' + str(epoch) + ".pth"
        torch_path = r'./models/model_' + str(epoch) + ".pth"
        torch.save(save_file, torch_path)
        
    f.close()  # close the recording text file
    return txt_name

import re


def get_loss_dice(txtpath):
    """
    From the record, get the training loss and validation dice.
    :param txtpath: record text file path
    :return: training loss and validation dice
    """

    f = open(txtpath, 'r')  # open the file, read-only
    train_loss = []  # variable for saving training loss
    val_dice = []   # variable for saving validation loss
    
    # read the text file line by line
    for line in f:
        current_train_loss = re.findall(r'train loss=(.*?),', line)[0]  # get the train loss data
        train_loss.append(float(current_train_loss))
        
        current_val_dice = re.findall(r'val dice=(.*?),', line)[0]  # get the validation data
        val_dice.append(float(current_val_dice))
    f.close()  # close the text file
    return train_loss, val_dice

2.9.3 Epoch Number¶

Epoch number is an important hyperparameter while training. An epoch refers to the process of sending all data into the network to complete a forward calculation and back propagation. Epoch number determines the learning iteration times, which directly affects the learning effect. Too large number wastes too much training time and may lead to an over-fitting, while too small value could not guarantee the good learning result. Here we train the whole dataset for totally 200 epochs, and record the training and validation log for each epoch.

def rle_encoding(x):
    '''
    *** Credit to https://www.kaggle.com/rakhlin/fast-run-length-encoding-python ***
    x: numpy array of shape (height, width), 1 - mask, 0 - background
    Returns run length as list
    '''
    dots = np.where(x.T.flatten() == 1)[0]
    run_lengths = []
    prev = -2
    for b in dots:
        if (b > prev + 1): run_lengths.extend((b + 1, 0))
        run_lengths[-1] += 1
        prev = b
    return run_lengths

2 Implementation¶

In this part, the structure of Segnet, FCN and deeplabv3 will be introduced. Besides, some loss functions and optimizers will also be described, and it will show the optimal options.

2.1 Network¶

Three network models are built in this project, which are SegNet, FCN and DeepLabV3, and DeepLabV3 is selected as the most suitable one.

2.1.1 SegNet¶

SegNet, an open resource project of image segmentation, is developed by a team of Cambridge University. Segmentation can be divided into two main parts: encoder and decoder[1]. An encoder is derived from VGG16 network model[2], which is mainly for the analysis of the information that objects contain, while the decoder is to map the analyzed information with the final images, which means that every pixel has a colour or label to show its object information. The encoder is a sequence of convolutional networks, which mainly consist of convolutional layers, pooling layers, and Batch-Normalization layers. Convolutional layers are for the capture of the local features of the image. Pooling layers are for the subsampling and the transformation of the same-size feature to the next layer. Batch-Normalization layers are for the distribution normalization of the training images, which accelerates the speed of learning. Encoder classifies and analyzes pixels of low-level local features to obtain high-level grammar meanings, while decoder collects these grammar meanings and labels same objects with their pixels to make all objects have their colours. Therefore, as the process of encoder obtains the approximate location information of all the objects, the decoder handles the process of labelling these objects with specific pixels. Decoder upsamples the minimized feature images, and then take actions of convolution with processed images to optimize geometric shapes of objects and to make up for the loss of details caused by the minimization of the pooling process[3]. However, due to the decoder structure in the Segnet, we expect that the inference time would take longer than the other two networks. There is one way to avoid the enormous number of parameters in the segnet, which is combining the squeeze net with the Segnet. It is also knowns as squeeze segnet[4]. It can significantly shorten the inference time in the segnet by making use of the advantages of the squeeze net. Therefore, we try to use this combination in our experiment section.

2.1.2 FCN and DeepLabV3¶

Both FCN and DeepLabV3 have Resnet as the backbone. Resnet is divided into five stages[5]. Stage 0 is regarded as the preprocessing of input, and the last four stages are composed of bottlenecks[6]. Stage 1 contains three bottlenecks, and the remaining three stages include 4, 6 and 3 bottlenecks, respectively. After FCN is sampled by the previous stage of convolution, the deconvolution layer is used to sample the last feature map obtained by the convolution to restore it to the size of the input image. FCN classifies each pixel while retaining the spatial information of the original input image. Finally, the SoftMax classification loss is calculated through it pixel by pixel. FCNHead connects Layer3 and Layer4 and adjusts the channel value of the input feature layer or feature matrix through a 3*3 convolutional layer, reducing it to 1/4 of the original[6]. Finally, after dropout and a 1*1 convolutional layer, the channel of the feature layer is adjusted to the number of categories in the segmentation task. Bilinear interpolation restores the image to its original size. The auxiliary classifier prevents the error gradient from propagating to the shallow layers of the network.
The structure of DeepLabV3 with Resnet backbone has an initial block, four layers, ASPP and the final block[7]. The input image passes through the initial block in 7*7 convolution and max pooling, and the output size is reduced by four times. About the four layers, the first two layers are the same as Resnet, and the latter two layers add astrous convolution. The astrous convolutions for layer 3 use fixed dilation but different dilations for layer 4 and the ASPP module. The astrous convolutions are present serially in layer 3 and layer 4 but parallelly in the ASPP module[8]. ASPP module includes five branches, one 1*1 convolution branch, three astrous convolution branches and one image pooling with 1*1 convolution branch. It contacts five branches and gets the result of the ASPP module. For the final block, there are two convolution layers and an up-sampling layer, which interpolates the number of classes of feature maps to the size of the input image using bilinear methods. DeepLabV3 has two main challenges in applying deep convolutional neural networks for semantic segmentation tasks[9]. The first challenge is that continued pooling and downsampling will reduce the resolution of features, which makes localization difficult. The second one is the existence of multi-scale targets. For the first challenge, DeepLabV3 proposed perforated convolution to improve the resolution of features while maintaining parameters and calculations. For the second challenge, DeepLabV3 proposed a multi-scale cascade or planetary convolution to improve the ASPP (Atrous Spatial Pyramid Pooling) module. Besides, the expanded convolutional layer with a large sampling rate will degenerate into a 1*1 convolutional layer when it cannot capture long-distance features due to the image boundary response. DeepLabV3 fuses the image-level features into the ASPP module to obtain the global context. In conclusion, DeepLabV3 encodes image-level global context features and further improves performance. DeeplabV3 is much better than the previous Deeplab, without post-processing of sense CRF, and has achieved state of art performance on the Pascal VOC 2012 semantic image segmentation benchmark[9]. For the neural network architectures, we decide to use DeepLabV3 as our final model, because after our experiments, we find that the dice of DeepLabV3 after all models converge is highest, which means that the performance of this model is best. At the same time, the running time and GPU memory used of it are also only slightly above these of e segnet. These will be discussed more in the Experiment module.

2.2 Loss Function¶

In machine learning, the difference between the predicted value of the samples and the true value of the model is usually called loss. The smaller the loss, the better the model, and the function used to calculate the loss is called the loss function. After talking about networks, we need to choose a suitable loss function for our neural network. Here we introduce four kinds of loss function: Cross-Entropy, Dice Loss, Focal Loss, and Boundary Loss and talk about features for each of them.

2.2.1 Cross-Entropy¶

Cross-entropy can be used in most segmentation situations[10]. Compared with other loss functions, it has two significant advantages: 1. It can measure subtle differences 2. Convex optimization function, which is convenient for using the gradient descent method to find the optimal solution. but its main drawback is that when it comes to segmenting the foreground and background, in another word the amount of foreground pixels is extremely less than that of background pixels, the model will deeply depend on the background, which leads to bad training results. To avoid that, we chose to use Cross-Entropy and Dice in our work, which works better than using them singly.

2.2.2 Dice Loss¶

Dice Loss is a function to measure the similarity of sets, usually used to compute the similarity degree of two samples[10]. For binary classification, Target segmentation images only have two values, therefore, it is efficient to ignore background pixels. Focused on the target parts that are being detected, the more similarities prediction results with true labels have, the higher the Dice coefficient will be, and the smaller the Dice Loss will be. Meanwhile, Dice Loss works well when target samples are extremely unbalanced. Though Dice Loss will be largely due to the unsuccessful segmentation when targets are small, which has a negative influence on the Back Propagation and makes the training unstable, it performed pretty well as our loss function. Therefore, we also used Cross-Entropy in our work to get better performance.

2.2.3 Focal Loss¶

Focal Loss was introduced in the paper written by the team of HeKaiming, which was to solve the problem caused by the unbalance of the hard samples and easy samples[11]. In one-stage objects detection, the negative samples and positive samples are heavily unbalanced, thus the loss function heavily decreases the proportion of the amount of simple negative samples in the training process, which also can be comprehended as difficult sample mining. For positive samples, the higher the probability of the output gets, the smaller the loss becomes. For negative samples, the lower the probability of the output gets, the smaller the loss becomes. To solve the unbalanced problem of negative samples and positive samples, Focal Loss chose to minimize the loss of samples with high confidence. However, its training results scored less than Dice loss in our work.

2.3 Optimizer¶

After getting loss function, our task begins to minimize it. The process of minimizing loss is so-called optimization. Here we introduce some usual optimizers and compare them from various aspects.

2.3.1 SGD¶

SGD has no conception of momentum, and its main drawback is that its slow descent speed, and it may continuously oscillate on both sides of the gully, staying at a local optimum[12]. Meanwhile, it is difficult to select the learning rate properly. Therefore, we did not choose this method for our work.

2.3.2 Adam¶

Adam can be seen as the combination of RmSprop and Momentum. Like RMSprop which uses exponential moving average for second-order momentum, Adam also uses exponential moving average for first-order momentum[13]. Its core philosophy is simple and straight, since the square of the nearest gradient shows the information we gain from each weight, we can divide it by its point to make sure that even the weight with the slowest change can make contributions.

2.3.3 AdamW¶

To use AdamW, we need to understand weight attenuation and L2 regularization. L2 regularization is a classic method for reducing over-fitting[14]. It adds a penalty term which consists of the sum of squares of all weights of the model to the loss function and multiplies with a specific hyperparameter to control the penalty intensity. For the ladder function in the optimizer, we only need to use the gradient to correct the parameter and not use the value of the parameter itself at all. As for weight decay, we can simply implement the weight attenuation before the optimizer[15]. Therefore, we decided to choose AdamW as our optimizer.

3.5 Hyperparameter¶

We could use different learning rates, batch sizes, epoch numbers, and other different hyperparameters during training. Here we try separate ways and values with respect to each of them and compare the dice score performance on the validation set.

3.5.1 Batch-Size¶

An epoch refers to the process of sending all data into the network to complete a forward calculation and backpropagation. Since an epoch is often too large for the computer to load, we will divide it into several smaller batches, while the batch size is the number of training samples in each batch. It could determine the speed and accuracy of convergence. Too small value leads the speed of convergence too slow, while a too-large value may disturb the direction and accuracy of convergence. What else we need to consider is memory efficiency and memory capacity and find the best balance between them. Here we try different values from 2 to 10 for batch size and show the validation result for each of them as Figure 1. As we can see, when a batch size is a small number, it is hard to converge in a limited epoch number. When a batch size is a considerable number, the number of iterations required to run an epoch is reduced, which means less time is needed for training. However, the iteration it takes is increased to achieve the same accuracy for the neural network. We could find that when batch size is equal to 6, the convergence speed and result reach a relatively good point, while the training efficiency is also acceptable.

3.5.2 Learning Rate¶

As an important hyperparameter in deep learning, the learning rate determines whether the objective function can converge to a local minimum and when to converge to the minimum. A suitable learning rate can make the objective function converge to a local minimum in a suitable time. Here we try the different initial learning rates from 0.00001 to 1 for optimizer and show the validation result for each of them. As we can see in Figure 2, a too-large learning rate could lead to non-convergence, while a too-small learning rate will converge to a local minimum, which leads to a bad learning effect. We could see that 0.001 is an appropriate initial learning rate for training our model, which has appropriate convergence speed and result.

3.5.3 Epoch Number¶

The epoch number is an important hyperparameter during training. Epoch number determines the learning iteration times, which directly affects the learning effect. A too substantial number wastes too much training time and may lead to an over-fitting, while too small value could not guarantee a good learning result. Here we train the whole dataset for 200 epochs and record the training and validation log for each epoch as Figure 3 shows. As we can see, as the epoch number increases, the training loss keeps going down but changes from steep to gentle, finally coming to convergence, while the validation dice score has already converged at around 80 epochs. As a result, for the model we selected and this training set, 80 epochs are enough for training to get the expected result.
